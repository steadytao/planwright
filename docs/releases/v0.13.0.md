This release makes Planwright's compatibility claims easier to audit. It adds fixture-backed checks for the current local command surfaces and tightens release evidence around signed checksums, GitHub attestations and release-attached SLSA provenance.

# Highlights
- Added a compatibility fixture runner and metadata model so public support claims are tied to checked examples.
- Added canonical, unsupported and malformed fixtures across the current local input surfaces.
- Added release-attached SLSA provenance as `planwright.intoto.jsonl` alongside the existing signed checksum and GitHub attestation model.
- Added SARIF fixture output validation for supported SARIF 2.1.0 output.

# Security
- No known security fixes in this release.

# In This Release
- Added fixture metadata for source format, source kind, compatibility level, expected loss categories and command expectations.
- Added fixture-backed checks for the compatibility matrix so rows must name real fixture evidence or an explicit documentation-only marker.
- Added fixtures for the AWS web application proof path, public database refusal, Terraform plan review, CloudFormation, SAM, Kubernetes/Gateway/Cilium, AWS scan bundles, malformed inputs and unsupported constructs.
- Added SARIF validation for fixture outputs that declare `want_sarif_files`.
- Documented the fixture authoring and golden-output update rules.
- Reworked the post-v0.12 roadmap so v0.13 proves current compatibility before Terraform/OpenTofu state and provider schema work.
- Split later Terraform/OpenTofu roadmap gates into state import, provider schema evidence and graph lowering.
- Narrowed the v1.0 target to stable core guarantees rather than broad feature expansion.
- Documented release provenance verification for OpenPGP checksum signatures, GitHub artefact attestations and release-attached SLSA provenance.
- Removed the unpinned SLSA reusable workflow path so third-party GitHub Actions dependencies remain SHA-pinned.
- Updated GitHub Actions pins to the latest checked SemVer tags.
- Updated the Go toolchain pin to `go1.26.4` for standard-library vulnerability fixes reported by `govulncheck`.
