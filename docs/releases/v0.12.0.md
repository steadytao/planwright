This release makes Planwright easier to understand, run and review through one polished proof path. It focuses on evidence quality and documentation clarity rather than broadening compatibility.

# Highlights
- Added a canonical AWS web application walkthrough that shows validation, security notes, cost notes, Terraform/OpenTofu-oriented review files, Mermaid output and a Planwright pack from one typed plan.
- Tightened the README and roadmap around proof before expansion so new users see the supported path before the larger command catalogue.
- Added regression coverage for the full proof path and the public-database risk case so public documentation stays tied to current CLI behaviour.

# Security
- No known security fixes in this release.

# In This Release
- Added `examples/aws-webapp-basic/README.md` as the canonical v0.12 walkthrough.
- Added a top-level README quick proof path before the full command list.
- Added a top-level README non-goals section that makes clear Planwright is not a one-click deployer, universal converter, compliance tool, live cloud scanner or drag-and-drop cloud diagrammer in the current implementation.
- Added ADR 0013 to record the proof-before-expansion decision.
- Updated the roadmap so v0.12 is a usability and proof release rather than a feature-surface expansion.
- Added CLI test coverage for the full AWS web application proof path.
- Added CLI test coverage for public database risk reporting and Terraform/OpenTofu generator refusal.
