This release is Planwright's first planned published release. It gathers the engine, importer, review, local web, governance and release-readiness work that existed before the project started using formal release tags.

Earlier version gates from `v0.0.1` through `v0.10.0` are internal project-history milestones, not published releases.

# Highlights
- Local-first infrastructure planning through a Go CLI, typed plans and `planwright.graph.v1`.
- Loss-aware import and review paths for selected CloudFormation, SAM, Terraform plan JSON, Kubernetes manifest and local AWS scan bundle inputs.
- Release-ready repository hygiene with CI, linting, spelling, documentation style, release notes, platform icon packaging, checksums, release SBOMs, human-controlled OpenPGP checksum signing and ADR-backed project boundaries.

# Security
- No known security fixes in this release.
- The local web workbench is loopback-only by default, rejects unexpected Host headers and does not read credentials or contact cloud APIs.
- Policy profiles and review outputs are static local evidence. They are not compliance certification and do not authorise deployment.

# In This Release
- Added `planwright.v1` typed plan parsing, graph lowering, validation diagnostics and the first AWS web application example.
- Added Terraform/OpenTofu-oriented generated review files, Mermaid output and Markdown security, cost, deployability, cleanup and assumptions reports.
- Added directory-based Planwright pack output.
- Added CloudFormation and SAM subset import with graph JSON output and Markdown loss reports.
- Added Terraform plan JSON review with Markdown and SARIF output.
- Added rendered Kubernetes, Gateway API and Cilium manifest import for the supported subset.
- Added local AWS scan bundle import for selected AWS CLI JSON artefacts.
- Added graph diff review, graph schema export, graph JSON validation and built-in policy profile review.
- Added a loopback-only text-and-table browser workbench through `planwright serve`.
- Added GitHub Actions workflows for build, tests, lint, static analysis, vulnerability checks, module hygiene, documentation style, workflow validation and release verification.
- Added governance, support, security, DCO, ADR, changelog, contributor and release documentation.
- Added direct release binary naming, generated Windows executable icon resources, Linux desktop metadata packages, macOS app bundles, release-level SPDX and CycloneDX SBOM preparation and OpenPGP-signed checksum manifest preparation for the planned first published release.
